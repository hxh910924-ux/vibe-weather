# 产品需求文档（PRD）

> 项目名称：氛围天气
> 版本：v1.0
> 状态：✅ 已确认

---

## 一、产品概述

氛围天气是一个简约沉浸式天气展示网页应用。它不追求数据堆砌，而是通过实时天气驱动的视觉动效（渐变、雨滴、雪花、云雾等）让用户"感受到"天气。页面以全屏氛围动效为背景，天气信息以极简方式叠加展示。支持自动定位和手动切换城市、日期，也可随时手动切换天气主题来体验不同的氛围效果。

目标用户：追求美学体验、喜欢沉浸式界面的日常用户。

---

## 二、核心功能需求

### 功能1：天气数据展示
- **描述**：在页面中心区域以极简方式展示当前天气的核心信息
- **用户操作**：打开页面自动加载，切换城市或日期后自动更新
- **预期结果**：显示以下信息，字体大而通透，融入背景：
  - 温度（大字号居中）
  - 降水情况（小雨/中雨/大雨/无雨）
  - 日出日落时间
  - 风力信息（风向 + 风力等级）
  - 当前城市名（小字，页面顶部偏上位置）

### 功能2：天气氛围动效
- **描述**：全屏背景根据天气状况渲染不同的视觉动效，营造沉浸感
- **用户操作**：动效随天气自动切换，也可手动选择主题
- **预期结果**：
  - **晴天**：明亮通透的暖色渐变动画，光线流动感
  - **多云**：柔和灰蓝渐变，缓慢飘动的云层
  - **雨天**：深蓝灰底色 + 毛玻璃雨滴效果，雨滴从上方滑落
  - **雪天**：冷色调渐变 + 白色颗粒飘落效果
  - **雾天**：多层半透明模糊层缓慢移动
  - **夜间**：深色星空背景，星光闪烁
  - 各动效之间切换时，使用 1-2 秒的淡入淡出过渡

### 功能3：手动主题切换
- **描述**：提供一组天气主题供用户随时手动切换，不依赖实际天气
- **用户操作**：点击页面底部或侧边的半透明主题切换入口（小圆点或小图标），选择后动效立即切换
- **预期结果**：
  - 主题入口常驻但不抢眼（半透明、小尺寸、悬停才显示标签）
  - 可选主题：☀️ 晴天 / ⛅ 多云 / 🌧 雨天 / ❄️ 雪天 / 🌫 雾天 / 🌙 夜间
  - 切换后天气数据不联动，纯视觉体验

### 功能4：城市选择
- **描述**：默认使用浏览器定位获取当前城市天气，支持手动搜索切换
- **用户操作**：
  - 点击页面顶部的城市名，弹出一个轻量搜索框
  - 输入城市名后显示搜索结果列表，点击选择
- **预期结果**：
  - 首次打开自动请求定位权限
  - 搜索框为毛玻璃风格弹窗，不打断沉浸感
  - 最近选择过的城市保存在 localStorage，下次打开优先显示

### 功能5：日期选择
- **描述**：支持查看未来一周（含今天）的天气预报
- **用户操作**：
  - 在温度数字下方，显示一排日期（今天 ~ +6天），用小圆点或横线分隔
  - 点击某天，天气数据和动效平滑过渡到对应日期
- **预期结果**：
  - 当前选中日期有高亮指示（微光或下划线）
  - 日期选择栏半透明融入背景，不占视觉重心
  - 最多显示 7 天，不做日历翻页

## 三、交互流程

```
打开页面
  → 浏览器请求定位权限
  → 获取当前位置 → 调用天气API获取数据
  → 根据天气状况渲染对应氛围动效
  → 展示温度、降水、日出日落、风力
  → 展示日期选择栏（今天 ~ +6天）
  → 展示底部主题切换入口（半透明小图标）

用户点击城市名
  → 弹出毛玻璃搜索框
  → 输入城市名 → 显示搜索结果
  → 点击选择 → 天气数据和动效平滑过渡

用户点击某天日期
  → 天气数据和动效平滑过渡到该天

用户点击底部主题图标
  → 动效立即切换到对应主题（数据不联动）
```

---

## 四、数据与存储

- **存储方式**：localStorage
- **数据内容**：
  - 最近选择的城市列表（最多 5 个）
  - 当前选中的主题（如有手动切换）
  - 是否已授权定位
- **数据格式**：JSON
- **天气数据**：通过免费天气 API 实时获取，不缓存（或缓存 30 分钟）

---

## 五、边界条件与限制

- 纯前端实现，无需后端服务
- 使用免费天气 API（如 OpenWeatherMap 免费版、和风天气免费版等）
- 所有依赖通过 CDN 引入，不需要 npm
- 页面需适配手机竖屏和电脑横屏
- 定位失败时，默认显示北京天气并提示用户手动选择城市
- API 请求失败时显示优雅的错误提示（融入背景风格），不显示报错堆栈

---

## 六、不在本版本范围内

- 不做历史天气查询
- 不做小时级逐时预报
- 不做空气质量（AQI）、湿度、气压等额外数据
- 不做 widget/桌面小组件
- 不做 PWA / 离线缓存
- 不做用户账号体系

---

## 七、开发工具

- **开发工具**：Trae
- **运行方式**：本地启动服务（Trae 内置预览或本地 HTTP 服务器）
